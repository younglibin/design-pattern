1.研磨设计模式--工厂方法
	用户创建一个接口，让子类决定实例化哪一个类，工厂方法将类的实例化延迟到子类

一、注意问题：
	1.暴露给客户端的接口不变
	2.调用接口的额实现类是多个，
	3.实现类的实例化在器子类中实现
二、实现思路：
	 1.编写客户端使用的接口方法
	 2.分别实现多个实现类，不同的实现类，实现的操作方式不同
	 3.编写一个，调用不同实现类的 主方法，在该方法中，提供一个获得 不同实现类对象的缺省方法，
	 		该方法在不同子类中做具体实现，
	 4.在 子类中 根据 不同的实现方式 返回对应的 实现类对象，供父类调用
	 
	 5.客户端调用，
	 		在客户端 缺省方式 的不同子类 来 决定进行哪种操作！
	 		
	 		
三、使用到的技术：
	java 的继承  、多态


四、使用场景：
	针对一个需求可能有多重处理方式，根据实际需要 动态支持 不同的处理方式
