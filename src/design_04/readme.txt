设计模式--桥连接模式
			将抽象部分和实现部分分离使他们可以独立的变化
			
			
			例如处理一个问题 有两个维度 
			例如：
			发消息：
			维度1：发送方式：短信，邮件，站内信息
			维度2：发送级别：普通，加急，即时
			
			把这两个维度分开 
			
			
			如果是使用IOC 这样的方式  ，使用这种方式，只需要在 配置文件中 修改注入的类 即可 

一、注意问题： 
			在这里 注意  再设计的时候一定要划分清楚 维度 

二、实现思路： 
			将两个不同的维度 分别实现
			1.例如发消息：总的接口 就是 发送消息
			2.用什么发 是一个维度
			3.发那种类型消息 又是一个维度 
			
			1.在用什么发消息的父类中  获得一个 某种发送类型；（本例的实现）
			2.在某种发送类型中 获得一个 用那舍呢么发 ；
			
			这里到底选择 谁做接口 ，谁做  缺省类  ，主要看 维度划分  ，和以后的扩张  综合考虑
	
三、使用到的技术： 
		java 的 继承 ，多态  

四、使用场景： 
		处理一个问题 有多种组合处理方式，将多种组合处理方式分为 两部分 
		，将其中一种处理维度 ，注入到另一种处理维度中 ！
